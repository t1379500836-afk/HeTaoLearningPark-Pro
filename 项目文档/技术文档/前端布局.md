## 二、响应式设计

### 断点规范

| 断点 | 屏幕宽度 | 设备类型 |
|------|----------|----------|
| `--breakpoint-xs` | 480px | 手机小屏 |
| `--breakpoint-sm` | 768px | 手机标准 |
| `--breakpoint-md` | 1024px | 平板 |
| `--breakpoint-lg` | 1200px | 桌面 |

### 媒体查询使用

```css
/* 手机优先 */
.component { font-size: 14px; }

/* 平板及以上 */
@media (min-width: 768px) {
  .component { font-size: 16px; }
}

/* 桌面 */
@media (min-width: 1024px) {
  .component { font-size: 18px; }
}
```

### 触控优化

| 要求 | 值 |
|------|-----|
| 按钮最小尺寸 | 44×44px |
| 避免悬停依赖 | 确保所有功能可通过触控完成 |

### 响应式导航菜单（移动端）

| 断点 | 菜单宽度 | 说明 |
|------|---------|------|
| ≤768px | 55% | 从右侧滑入 |
| ≤480px | 60% | 优化触控体验 |

移动端采用汉堡菜单设计，点击后菜单从屏幕右侧滑入。

## 四、课程体系页面布局

**文件**: `src/views/CourseLevelsView.vue`

采用**横向轮播式布局**：图形化 → Python → C++

```
┌─────────────────────────────────────────────────────┐
│  Hero 区域（渐变背景）                                │
│  图形化 → Python → C++                               │
│  ┌──────┐ ┌──────┐ ┌──────┐                         │
│  │🌱PY1│ │🌿PY2│ │🌳PY3│  ← Python阶段入口         │
│  └──────┘ └──────┘ └──────┘                         │
├─────────────────────────────────────────────────────┤
│  轮播区域                                            │
│  ◄  ┌─────────┐ ┌─────────┐ ┌─────────┐  ►         │
│     │ 图形化  │ │ Python  │ │  C++    │             │
│     │ (侧边) │ │ (中心)  │ │ (侧边) │             │
│     └─────────┘ └─────────┘ └─────────┘             │
│                                                     │
│     [图形化]  [Python]  [C++]   ← 底部指示器        │
└─────────────────────────────────────────────────────┘
```

### 课程卡片配色

| 卡片 | 顶部渐变条 | 主题色 |
|------|-----------|--------|
| 图形化 | 绿色渐变 `#4cd964 → #34c759` | 🟢 入门推荐 |
| Python | 橙色渐变 `#ff9f43 → #ff6b6b` | 🟠 当前学习 |
| C++ | 蓝紫渐变 `#5856d6 → #007aff` | 🔵 进阶挑战 |

### 响应式布局

| 断点 | 卡片宽度 | 布局特点 |
|------|---------|----------|
| > 1200px | 380px | 三卡片可见，Python居中 |
| 1024px | 320px | 卡片缩小 |
| 768px | 280px | Python居中优先 |
| 480px | 260px | 阶段入口紧凑 |

### 容器布局规范

| 属性 | 值 | 说明 |
|------|-----|------|
| `width` | `100%` | 确保占满父元素宽度 |
| `max-width` | `1000px` (桌面端 `1100px`) | 限制最大宽度 |
| `margin` | `0 auto` | 水平居中 |
| `box-sizing` | `border-box` | 包含padding计算宽度 |
| `padding` | `20px` + `padding-bottom: 100px` | 留出底部空间 |

### 卡片网格布局

| 断点 | grid-template-columns | 布局 |
|------|----------------------|------|
| > 768px | `repeat(auto-fit, minmax(300px, 1fr))` | 两列并排 |
| ≤ 768px | `1fr` | 单列堆叠 |

## 八、设计原则

1. **移动优先**: 从小屏幕开始设计，渐进增强
2. **一致性**: 保持不同设备间的视觉风格和交互一致性
3. **可访问性**: 确保所有用户都能使用
4. **性能**: 优化加载速度和渲染性能
