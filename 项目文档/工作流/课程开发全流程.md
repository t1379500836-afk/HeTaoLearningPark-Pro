## 一、正课开发流程（PY1/PY2/PY3）

### 🚨 禁止臆造课程内容！

> **重要原则**：**以 OCR 提取内容为基础，合理拓展，保持课程一致性。**

| ❌ 错误做法 | ✅ 正确做法 |
|-----------|-----------|
| 完全根据课程主题自己编写单词、知识点 | 从 `content.json` 中提取 OCR 识别的内容作为核心 |
| 参考其他课程"复制"不相关的内容 | 在 OCR 内容基础上拓展**本节课相关**的内容 |
| 改变 OCR 提取的核心知识点含义 | 保持核心知识点不变，按三梯度标准展开 |
| 编写与课程主题无关的习题 | 参考 OCR 题目风格，根据知识点出**相关**习题 |

### 流程图

```
① 文件整理 → ② 内容提取(OCR) → ③ 数据开发
```

### ① 文件整理阶段

**源文件位置**：`源文件/levelX/`

| 原始文件名 | 标准命名 | 说明 |
|-----------|---------|------|
| `LX-X单词卡.png` | `flashcard.png` | 英文单词卡 |
| `知识总结.png` | `knowledge.png` | 知识点图 |
| `LX-X讲义.pdf` | `lecture.pdf` | 课堂讲义 |
| `LX-X课后习题.pdf` | `exercise.pdf` | 练习题 |
| `LX-X课后习题解析.pdf` | `solution.pdf` | 答案解析 |

**目标位置**：`src/data/courses/PX/LX/LX-*/`

**操作步骤**：
1. 筛选最新版本（优先"新logo版"）
2. 按上表重命名为标准名称
3. 复制到目标目录

### ② 内容提取阶段

```bash
# 安装依赖（首次）
pip install -r scripts/requirements.txt

# 运行提取脚本
python scripts/extract_content.py

# 指定阶段（可选）
python scripts/extract_content.py PY1
```

**输出文件**：
- `src/data/courses/PX/content.json` — 结构化数据
- `src/data/courses/PX/OCR-CONTENT.md` — 可读参考文档

### ③ 数据开发阶段

**创建文件**：`src/data/courses/PX/lessons/LX-*/lesson-data.js`

**参考模板**：[技术文档/课程数据结构.md](../技术文档/课程数据结构.md)

**数据来源**：从 `content.json` 中提取对应课次内容

**规则说明**：
- ✅ **单词卡**：4-6个，OCR提取的必须包含，至少拓展1个
- ✅ **知识点**：从OCR提取的核心知识点出发，按三梯度标准编写
- ✅ **习题**：**6道**（2基础+2进阶+2挑战），参考OCR题目风格

**禁止事项**：
- ❌ 不能跳过OCR提取直接编写
- ❌ 不能参考其他课程"复制"内容
- ❌ 不能改变OCR提取的核心知识点

## 二、YCL专区开发流程

### ⚠️ 核心原则（必读）

> **考情分析驱动**：严格按照 `exam-analysis.js` 中的必考知识点出题！
>
> **审查必做**：出完试卷必须进行知识点覆盖和 distribution 统计审查！

详细规范见：[YCL试卷开发规范](../../src/data/courses/YCL/README.md)

### 开发流程图

```
① 阅读考情分析 → ② 设计试卷结构 → ③ 编写题目 → ④ 审查验证 → ⑤ 更新统计
```

### ① 阅读考情分析

必读文件：`src/data/courses/YCL/levelX/exam-analysis.js`

重点关注：
- `examStatus.coding === '必考'` 的知识点
- `scoreRange` 分值范围
- `importance` 重要程度

### ② 设计试卷结构

编程题固定结构（55分）：

| 题号 | 分值 | 建议考查点 |
|------|------|------------|
| 第1题 | 10分 | 简单计算（数学运算 + input/int） |
| 第2题 | 10分 | 字符串处理（字符串拼接） |
| 第3题 | 15分 | 条件判断（if-else） |
| 第4题 | 20分 | 循环结构（for循环） |

### ③ 编写题目

- 每道题必须标注正确的 `knowledgePoint`
- 题目难度要与 `difficulty` 一致
- 分值要符合标准

### ④ 审查验证

**必查项目**：

1. **知识点覆盖**：所有必考知识点是否都有编程题？
2. **distribution 准确**：统计数据是否与实际题目一致？
3. **分值正确**：单选2分、多选3分、编程10+10+15+20？

### ⑤ 更新统计

修改题目后必须同步更新 `distribution` 对象！

### 题库设计

每个等级 **5套练习题**：

| 难度 | 套数 | 说明 |
|------|------|------|
| 基础 | 3套 | 与真题难度一致，覆盖必考考点 |
| 进阶 | 1套 | 略微提升难度，增加综合题 |
| 提升 | 1套 | 多知识点综合，接近考试上限 |

### 题型分布

| 题型 | 数量 | 分值 |
|------|------|------|
| 单选题 | 15题 | 30分（2分/题） |
| 多选题 | 5题 | 15分（3分/题） |
| 编程题 | 4题 | 55分（10+10+15+20） |

### 题目生成原则

> **重要**：所有练习题必须基于用户提供的资料生成，严禁超纲！

| 原则 | 说明 |
|------|------|
| 考点覆盖 | 覆盖所有必考知识点 |
| 难度控制 | 不超出真题难度范围 |
| 题型一致 | 题型与真题保持一致 |
| 分值匹配 | 分值分布与真题一致 |

## 三、YCL专区知识点核对流程

### 核对目的

确保 `exam-analysis.js` 中的知识点定义与原始题库 `temp-extracted/` 中的实际内容一致。

### 核对步骤

```
① 读取题库 → ② 搜索关键词 → ③ 对比知识点 → ④ 同步更新
```

### 步骤详解

#### ① 读取原始题库

```bash
# 题库位置
src/data/courses/YCL/temp-extracted/
├── level4-raw.txt
├── level5-raw.txt
└── level6-raw.txt
```

#### ② 搜索关键词统计

使用 Grep 工具搜索题库中的函数/方法出现情况：

| 知识点类型 | Grep 搜索模式 |
|-----------|--------------|
| 统计函数 | `\b(max|min|count|len|sum)\s*\(` |
| 列表方法 | `\b(append|remove|pop|insert|del)\s*\(` |
| 排序函数 | `\b(sort|sorted)\s*\(` |
| 循环控制 | `\b(break|continue)\b` |
| 类型转换 | `\b(int|str|list|float)\s*\(` |
| print参数 | `sep=\|end=` |

#### ③ 对比知识点定义

检查 `exam-analysis.js` 中的每个知识点：

```javascript
// 示例：核对统计命令
{
  id: 'kp-5-2',
  name: '统计命令',
  alias: ['max', 'min', 'len', 'sum'],  // ← 必须与题库实际出现的函数一致
  description: '使用max、min、len、sum等统计函数',  // ← 描述要准确
  reviewLocation: 'L7-2',  // ← 复习位置要正确
}
```

#### ④ 同步更新文件

**必须同时更新**：

| 文件 | 更新内容 |
|------|----------|
| `levelX/exam-analysis.js` | alias、description、reviewLocation |
| `config/theory.js` | 对应的 theory-5-2 等理论讲解内容 |

### 核对示例（五级统计命令）

**问题发现**：
- 题库中出现 `max()`、`min()`，但 alias 中没有
- alias 中有 `count`，但题库中 5 级没有 count()

**修正过程**：

```javascript
// exam-analysis.js 修正
alias: ['count', 'len', 'sum']  // ❌ 错误
alias: ['max', 'min', 'len', 'sum']  // ✅ 正确

// theory.js 同步修正
summary: '使用 count、len、sum 等函数'  // ❌ 错误
summary: '使用 max、min、len、sum 等函数'  // ✅ 正确
```

### 常见问题

| 问题 | 发现方法 | 修复方案 |
|------|----------|----------|
| alias 缺少函数 | Grep 搜索题库 | 添加到 alias 数组 |
| alias 多余函数 | Grep 搜索确认不存在 | 从 alias 移除 |
| description 不准确 | 阅读题库考点描述 | 更新 description |
| reviewLocation 错误 | 阅读题库复习位置 | 更新 reviewLocation |

## 四、YCL专区 reviewLocation 核对流程

### 核对目的

确保 `exam-analysis.js` 中的 `reviewLocation` 与正课内容匹配。

### 课程阶段映射

| YCL等级 | 对应阶段 | 课程范围 |
|--------|---------|---------|
| 四级 | PY1 | L1-1 ~ L6-3 |
| 五级 | PY2 | L7-1 ~ L12-3（部分基础在PY1） |
| 六级 | PY3 | L13-1 ~ L18-3（部分知识点在PY2） |

### 核对方法

1. **读取 lesson-data.js 头部注释**：确认每课核心知识点
2. **使用 Grep 搜索关键词**：验证知识点是否在对应课次中出现
3. **对比 reviewLocation**：检查格式是否正确（如 `L1-1` 而非 `L1`）
4. **修复或标记**：找到正确课次则更新，正课中不存在则标记为 `暂无`
