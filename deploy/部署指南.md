# HeTaoLearningPark-Pro 部署指南

从零开始，在一台全新服务器上完成全部部署。

---

## 一、安装宝塔面板

```bash
# CentOS
yum install -y wget && wget -O install.sh https://download.bt.cn/install/install_6.0.sh && sh install.sh ed8484bec

# Ubuntu / Debian
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh ed8484bec
```

安装完成后终端会显示：
- 宝塔面板地址（如 `http://IP:8888/xxxx`）
- 用户名和密码

登录宝塔面板，按提示安装 **Nginx** 和 **MySQL 5.7+**（编译安装或极速安装均可）。

---

## 二、安装 Node.js

在宝塔终端执行：

```bash
source /www/server/nvm/nvm.sh
nvm install 18 && nvm use 18 && nvm alias default 18
node -v   # 确认输出 v18.x.x
```

---

## 三、创建数据库

宝塔面板 → **数据库** → **添加数据库**：

| 字段 | 填写 |
|------|------|
| 数据库名 | `hetao_learning` |
| 用户名 | `hetao_learning`（默认与数据库名相同） |
| 密码 | 自己设置，记住它 |

---

## 四、上传项目文件

### 4.1 上传到服务器

通过宝塔 **文件管理器** 或 scp，将 `HeTao/` 目录下的全部内容上传到 `/www/wwwroot/hetao/`：

```
/www/wwwroot/hetao/
├── src/                       # 前端源码（构建必需，~2.4M）
├── public/                    # 静态资源（favicon.png）
├── index.html                 # Vite 入口 HTML
├── package.json               # 前端构建依赖
├── vite.config.js             # Vite 构建配置
│
├── dist/                      # 用户端构建产物
├── admin/dist/                # 管理后台构建产物
├── server/                    # 后端代码（不要上传 node_modules）
│   ├── index.js
│   ├── db.js
│   ├── package.json
│   ├── .env                   # 需要手动创建，见下一步
│   ├── middleware/
│   └── routes/
├── ecosystem.config.cjs       # PM2 配置
└── deploy/                    # 部署相关文件（可选）
```

> 注意：所有 `node_modules/` 都不要上传，在服务器上安装。

### 4.2 服务器安装依赖

```bash
# 前端构建依赖
cd /www/wwwroot/hetao
npm config set registry https://registry.npmmirror.com/
npm install

# 后端运行依赖
cd /www/wwwroot/hetao/server
npm install --omit=dev
```

### 4.3 服务器构建前端

```bash
cd /www/wwwroot/hetao
npm run build
# 产出 dist/ 目录，Nginx 即可提供服务
```

> 管理后台需在本地构建后上传 `admin/dist/`，服务端不包含管理后台源码。

---

## 五、配置后端

### 5.1 创建环境配置

创建 `/www/wwwroot/hetao/server/.env`，内容如下（根据实际情况填写）：

```env
# 服务端口
PORT=3000

# MySQL（填写第三步创建的数据库信息）
DB_HOST=localhost
DB_PORT=3306
DB_USER=hetao_learning
DB_PASSWORD=你设置的数据库密码
DB_NAME=hetao_learning

# JWT 密钥（随便填一个长字符串）
JWT_SECRET=hetao_learning_jwt_secret_2026
```

## 六、启动后端

```bash
cd /www/wwwroot/hetao
npm install -g pm2
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup    # 设置开机自启，按提示执行输出的命令
```

验证是否成功：

```bash
pm2 logs hetao-api --lines 10 --nostream
```

应看到：
```
数据库初始化完成
后端服务已启动: http://localhost:3000
```

## 七、配置 Nginx

### 7.1 添加站点

宝塔面板 → **网站** → **添加站点**：
- 域名：填写你的服务器 IP（如 `103.193.173.202`）
- 根目录：`/www/wwwroot/hetao`
- PHP：选 **纯静态**
- 点击提交

### 7.2 替换配置文件

点击刚创建的站点 → **设置** → **配置文件**，将内容全部替换为：

```nginx
server {
    listen 8080;
    server_name _;

    # 前端主站
    location / {
        root /www/wwwroot/hetao/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 管理后台
    location /admin {
        alias /www/wwwroot/hetao/admin/dist;
        index index.html;
        try_files $uri $uri/ /admin/index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    access_log /www/wwwlogs/hetao.log;
    error_log /www/wwwlogs/hetao.error.log;
}
```

保存后 Nginx 会自动重载。

## 八、放行端口

需要在 **两个地方** 都放行 8080 端口：

### 8.1 宝塔防火墙

宝塔面板 → **安全** → 放行 `8080`

### 8.2 云服务商安全组

登录云服务商控制台（阿里云/腾讯云/华为云等）：
- 找到该服务器的 **安全组**
- 添加入站规则：TCP 协议，端口 `8080`，源地址 `0.0.0.0/0`

## 更新部署

### 更新前端课程内容

```bash
# 1. 在本地修改 src/ 下的课程数据或组件
# 2. 上传新的 src/ 到服务器（覆盖）
# 3. 在服务器构建
cd /www/wwwroot/hetao
npm run build
# Nginx 自动提供新文件，无需重启
```

### 更新后端代码

```bash
# 1. 上传新的 server/ 到服务器（覆盖）
# 2. 如有依赖变化
cd /www/wwwroot/hetao/server && npm install --omit=dev
# 3. 重启后端
pm2 restart hetao-api
```

### 更新管理后台

```bash
# 1. 在本地构建
cd admin && npm install && npm run build
# 2. 上传 admin/dist/ 到服务器（覆盖）
# 无需重启，Nginx 直接提供新文件
```

### 教师数据自动更新流程

管理后台增删改教师时，后端自动处理：

1. 立即写入 MySQL 并更新 `src/config/teachers.config.js`
2. 标记构建任务
3. 每 10 分钟自动检查，有变更才执行 `npm run build`

无需手动操作，最多等待 10 分钟前端生效。
