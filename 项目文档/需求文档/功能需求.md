#### 题目生成原则

> **核心原则**：考情分析驱动，严格按 exam-analysis.js 出题，出完必审查！
>
> 详细规范见：[YCL试卷开发规范](../../src/data/courses/YCL/README.md)

| 原则 | 说明 |
|------|------|
| **考情分析驱动** | 严格按照 exam-analysis.js 中的必考知识点出题 |
| **完整覆盖** | 每套试卷必须覆盖所有必考知识点的编程题 |
| **审查必做** | 出完试卷必须进行知识点覆盖和统计审查 |
| **统计准确** | distribution 数据必须与实际题目一致 |

**编程题覆盖要求**（每套卷4题，55分）：

| 题号 | 分值 | 必考知识点 |
|------|------|------------|
| 第1题 | 10分 | 数学运算（+ input/int） |
| 第2题 | 10分 | 字符串拼接 |
| 第3题 | 15分 | if-else 条件判断 |
| 第4题 | 20分 | for 循环 |

**传统原则**：
- 必须基于用户提供的资料生成，严禁超纲
- 覆盖所有必考知识点
- 难度不超出真题范围
- 题型与真题保持一致

#### 各等级知识点范围

| 等级 | 已学内容 | 未学内容（禁止出现） |
|------|---------|---------------------|
| 四级 | print, input, int, 算术运算, if-else, for循环 | len(), 列表, 字典, 函数定义 |
| 五级 | 四级全部 + 索引, while, break/continue, 列表操作 | 自定义函数, 类, 文件操作 |
| 六级 | 五级全部 + 函数定义, Pygame, 递推算法 | 递归函数调用, 排序算法, 二分查找 |

### 8. 注意力训练模块

**页面**: `/attention`

#### 模块概述

注意力训练模块提供科学化的专注力训练游戏，帮助学员提升注意力、工作记忆和抑制控制能力。

#### 模块架构

```
注意力训练
├── 训练大厅（训练项目选择）
│   ├── 舒尔特挑战（视觉搜索）
│   ├── Stroop色词冲突（抑制控制）
│   ├── N-Back记忆（工作记忆）
│   └── Go/No-Go（冲动控制）
├── 训练统计（历史成绩、进步曲线）
└── 训练说明（科学原理、训练建议）
```

#### 训练项目详情

##### 8.1 舒尔特挑战（Schulte Grid）

| 功能 | 说明 |
|------|------|
| 网格规格 | 3×3 / 4×4 / 5×5 / 6×6 四档难度 |
| 核心玩法 | 按顺序点击 1→2→3...→N² 的数字 |
| 计时功能 | 精确到毫秒，完成后显示用时 |
| 视觉反馈 | 正确高亮绿色，错误高亮红色 |
| 难度递进 | 通关当前难度解锁下一难度 |
| 成绩记录 | 保存每个难度的最佳成绩 |

##### 8.2 Stroop 色词冲突（规划中）

| 功能 | 说明 |
|------|------|
| 游戏模式 | 判断文字颜色而非文字内容 |
| 难度分级 | 简单（色块）/ 中等（色词一致）/ 困难（色词冲突） |
| 训练目标 | 抑制控制能力 |

##### 8.3 N-Back 记忆（规划中）

| 功能 | 说明 |
|------|------|
| 游戏模式 | 判断当前刺激是否与N步前相同 |
| 难度分级 | 1-Back / 2-Back / 3-Back |
| 训练目标 | 工作记忆容量 |

##### 8.4 Go/No-Go（规划中）

| 功能 | 说明 |
|------|------|
| 游戏模式 | 特定信号快速点击，其他信号不点 |
| 训练目标 | 冲动控制、反应速度 |

#### 页面布局要求

| 元素 | 说明 |
|------|------|
| 入口位置 | 网站 Header 导航栏新增「注意力训练」入口 |
| 训练大厅 | 卡片式布局展示各训练项目 |
| 游戏界面 | 居中布局，移动端适配 |
| 结果页面 | 显示成绩、历史对比、再玩一次 |

#### 技术要点

| 项目 | 说明 |
|------|------|
| 数据持久化 | localStorage 保存成绩记录 |
| 随机算法 | Fisher-Yates 洗牌保证随机性 |
| 计时精度 | 使用 `performance.now()` |
| 动画效果 | CSS transition + animation |

## 三、功能限制

### Python编辑器 (Skulpt)

| 限制项 | 说明 |
|--------|------|
| 支持的标准库 | print、input、math、random、json、turtle 等 |
| 不支持的库 | numpy、pandas 等第三方库 |
| 特殊情况 | 在 `while True` 中使用 input() 可能导致无法停止 |

### 响应式设计

| 要求 | 说明 |
|------|------|
| 响应式断点 | 1024px, 768px, 480px |
| 设备支持 | 电脑、手机、平板、学习机 |
| 触控优化 | 按钮最小 44×44px |
