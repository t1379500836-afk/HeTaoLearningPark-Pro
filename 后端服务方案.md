# 后端服务方案

## 部署架构

```
服务器（宝塔面板）
├── Nginx
│   ├── /           → 学生端 dist/（静态文件）
│   ├── /admin/     → 管理后台 dist/（静态文件）
│   └── /api/       → 反向代理到 localhost:3000
└── Node.js 服务（Express，端口 3000）
    ├── API 路由
    ├── MySQL 数据库（宝塔自带）
    └── 数据变更时 → 生成静态 JS → npm run build
```

数据更新流程：管理后台操作 → Express 写 MySQL → 自动重新生成静态文件 → 自动 build → Nginx 直接提供新的 dist/

## 技术选型

| 技术 | 用途 |
|------|------|
| Node.js + Express | 后端 API |
| MySQL (mysql2) | 数据库（宝塔面板自带） |
| JWT + bcryptjs | 管理员认证 |
| Vue 3 | 管理后台前端 |

## 当前任务：搭建后端服务骨架

### 1. 后端项目初始化

在项目根目录创建 `server/` 目录：

```
server/
├── index.js          # Express 入口
├── db.js             # MySQL 连接 + 建表
├── middleware/
│   └── auth.js       # JWT 验证中间件
└── routes/
    ├── auth.js       # 登录接口
    └── teachers.js   # 教师口令 CRUD
```

### 2. 数据库表设计

**admins 表**（管理员账号）：
- id, username, password_hash, created_at

**teachers 表**（教师口令）：
- id, name（教师名称）, key（口令明文）, created_at

### 3. API 设计

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | /api/auth/login | 管理员登录 | 无 |
| GET | /api/teachers | 获取教师列表 | JWT |
| POST | /api/teachers | 新增教师 | JWT |
| PUT | /api/teachers/:id | 修改教师 | JWT |
| DELETE | /api/teachers/:id | 删除教师 | JWT |

### 4. 口令 CRUD 与静态再生

增删改教师后，后端自动：
1. 从 MySQL 读取全部教师数据
2. 重新生成 `src/config/teachers.config.js`（保持现有加密格式）
3. 执行 `npm run build`
