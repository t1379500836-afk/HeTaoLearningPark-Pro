# 后端服务方案

## 部署架构

```
服务器 /www/wwwroot/hetao/            ← 项目根目录
├── Nginx（端口 8080）
│   ├── /           → 学生端 dist/（静态文件，可接 CDN）
│   ├── /admin/     → 管理后台 admin/dist/（静态文件）
│   └── /api/       → 反向代理到 localhost:3000
├── Node.js 服务（Express，端口 3000，PM2 管理）
│   ├── API 路由
│   ├── MySQL 数据库（宝塔自带）
│   └── 数据变更时 → 生成配置 → 定时构建（每 10 分钟检查）
└── 构建工具链
    └── npm run build → 生成新 dist/ → Nginx 自动提供
```

## 服务器部署文件结构

上传到服务器 `/www/wwwroot/hetao/` 的完整文件结构：

```
hetao/
├── src/                         # 前端源码（构建必需，~2.4M）
│   ├── main.js
│   ├── App.vue
│   ├── router/index.js
│   ├── config/                  # 配置文件（含自动生成的 teachers.config.js）
│   ├── composables/             # Vue 组合式函数
│   ├── components/              # 组件（shared/ + course/）
│   ├── views/                   # 页面组件
│   ├── data/                    # 课程数据
│   └── assets/                  # 样式和图片
├── public/                      # 静态资源
│   └── favicon.png
├── index.html                   # Vite 入口 HTML
├── package.json                 # 前端依赖声明（构建用）
├── vite.config.js               # Vite 构建配置
│
├── dist/                        # 构建产物（Nginx 提供服务）
├── admin/                       # 管理后台
│   └── dist/
├── server/                      # 后端服务
│   ├── index.js                 # Express 入口
│   ├── db.js                    # MySQL 连接池 + 建表
│   ├── package.json             # 后端依赖
│   ├── .env                     # 环境变量（数据库密码、JWT 密钥）
│   ├── middleware/auth.js       # JWT 验证中间件
│   └── routes/
│       ├── auth.js              # 登录接口
│       └── teachers.js          # 教师 CRUD + 构建调度
├── deploy/
│   └── nginx.conf               # Nginx 配置模板
└── ecosystem.config.cjs         # PM2 配置
```

**注意：不要上传 `node_modules/`**，在服务器上分别执行：
```bash
cd /www/wwwroot/hetao && npm install        # 前端构建依赖
cd /www/wwwroot/hetao/server && npm install  # 后端运行依赖
```

## 首次部署步骤

```bash
# 1. 上传 HeTao 目录全部内容到服务器
scp -r HeTao/* root@你的服务器:/www/wwwroot/hetao/

# 2. 安装依赖
cd /www/wwwroot/hetao && npm install
cd /www/wwwroot/hetao/server && npm install

# 3. 配置环境变量
cp /www/wwwroot/hetao/server/.env.example /www/wwwroot/hetao/server/.env
# 编辑 .env 填入实际的数据库密码和 JWT 密钥

# 4. 启动后端服务
cd /www/wwwroot/hetao && pm2 start ecosystem.config.cjs

# 5. 配置 Nginx（参考 deploy/nginx.conf）
```

## 数据更新流程

```
管理后台操作（增/删/改教师）
  → Express 写入 MySQL（立即生效，API 已返回成功）
  → 立即重新生成 src/config/teachers.config.js
  → 标记 needsBuild = true
  → 每 10 分钟定时器检查：
      有标记 → 执行 npm run build → 生成新 dist/ → Nginx 自动提供
      无标记 → 跳过
```

**设计要点**：
- 配置文件每次操作立即更新，不丢失数据
- 构建采用脏标记 + 10 分钟轮询，避免连续操作触发多次构建
- 构建在后台异步执行，不阻塞 API 响应
- 1 核 1G 服务器上每次构建约 30-60 秒，10 分钟最多构建一次

## 技术选型

| 技术 | 用途 |
|------|------|
| Node.js + Express | 后端 API |
| MySQL (mysql2) | 数据库（宝塔面板自带） |
| JWT + bcryptjs | 管理员认证 |
| Vue 3 + Vite | 前端框架 + 构建工具 |
| PM2 | Node 进程管理 |

## API 设计

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | /api/auth/login | 管理员/教师登录 | 无 |
| GET  | /api/auth/me    | 获取当前用户信息 | JWT |
| GET  | /api/teachers   | 获取教师列表 | JWT |
| POST | /api/teachers   | 新增教师 | JWT（仅 admin） |
| PUT  | /api/teachers/:id | 修改教师 | JWT |
| DELETE | /api/teachers/:id | 删除教师 | JWT（仅 admin） |

## 数据库表设计

**teachers 表**：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主键 |
| username | VARCHAR(50) UNIQUE | 登录用户名 |
| password_hash | VARCHAR(255) | bcrypt 加密密码 |
| role | ENUM('admin','teacher') | 角色 |
| display_name | VARCHAR(100) | 显示名称 |
| key | VARCHAR(100) UNIQUE | 学生端口令 |
| created_at | TIMESTAMP | 创建时间 |
