# 后端服务方案

> 为核桃编程学习平台添加后端管理服务

---

## 一、功能清单

| 功能 | 类型 | 说明 |
|------|------|------|
| YCL 试卷上传 | 后端管理 | 通过管理界面上传/编辑/删除试卷 |
| 教师口令管理 | 后端管理 | 通过管理界面增删改查教师口令 |
| 班级表扬榜 | 后端管理 + 学生端展示 | 教师上传表扬内容，学生输入教师口令查看本班榜单 |
| 注意力训练 | 纯前端 | 舒尔特方格等专注力训练游戏 |

---

## 二、技术架构

### 核心思路：静态再生

所有数据更新都通过"静态再生"模式，学生端保持纯静态 + CDN，服务器零回源压力。

```
管理员/教师在后台修改数据
       ↓
后端写入 SQLite 数据库
       ↓
后端自动重新生成静态 JS 文件
       ↓
自动执行 npm run build
       ↓
新 dist/ 部署到 CDN

学生访问 → CDN 返回静态文件 → 浏览器本地运行（和现在完全一样）
```

- 学生端前端代码**零改造**（试卷加载、口令验证、表扬榜都走静态文件）
- 保持 CDN 架构，离线兼容
- 管理后台是独立 SPA，与学生前端完全分离

### 技术选型

| 技术 | 说明 |
|------|------|
| Node.js + Express | 后端框架 |
| SQLite (better-sqlite3) | 单文件数据库，零部署开销 |
| JWT (jsonwebtoken + bcryptjs) | 管理员认证 |
| Vue 3 | 管理后台前端 |

## 三、需求说明

### 3.1 YCL 试卷上传

- 管理后台支持上传/编辑/删除试卷（JSON 粘贴或文件上传）
- 上传前自动校验试卷格式（题目数量、分值一致性等）
- 修改后自动重新生成 `src/data/courses/YCL/*/sets/*.js` 文件并构建部署

### 3.2 教师口令管理

- 管理后台支持增删改查教师口令
- 修改后自动重新生成 `src/config/teachers.config.js` 并构建部署
- 口令仍为身份标识，沿用现有安全模型

### 3.3 班级表扬榜

- 教师在管理后台按班级上传表扬记录（学生姓名、表扬标题、描述、徽章）
- 学生端输入教师口令后展示对应班级的表扬榜
- 数据走静态再生模式：修改后自动生成静态数据文件并构建部署，不实时查询数据库

### 3.4 注意力训练

- 纯前端实现，不涉及后端
- 优先开发舒尔特方格（3×3 / 4×4 / 5×5 / 6×6 四档难度）
- 成绩保存到 localStorage
- 其余游戏（Stroop、N-Back、Go/No-Go）规划中
